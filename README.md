# APS - Análise e Projeto de Software

Este repositório tem como objetivo concentrar os documentos entregáveis da disciplina de APS (Análise e Projeto de Software).

O material reunido aqui registra os artefatos produzidos ao longo da disciplina, facilitando a organização, o versionamento e a consulta dos documentos do projeto.

## Objetivo

Centralizar, em um único lugar, os principais artefatos desenvolvidos na disciplina, tais como:

- documento de visão;
- documento de requisitos;
- documento de arquitetura de software;
- diagramas, modelos e demais entregáveis solicitados.

## Estrutura do Repositório

Atualmente, o repositório está organizado da seguinte forma:

```text
APS/
├── README.md
└── Documentos/
    ├── APS - Documento de Requisitos - SGFV - v1.1.docx
    ├── DocVisao.pdf
    └── Documento de Arquitetura de Software.docx
```

## Documentos Disponíveis

Os arquivos atualmente presentes no repositório são:

- `DocVisao.pdf`: documento de visão do projeto;
- `APS - Documento de Requisitos - SGFV - v1.1.docx`: documento de requisitos;
- `Documento de Arquitetura de Software.docx`: documento de arquitetura de software.

## Finalidade do Repositório

Este repositório serve para:

- manter os entregáveis centralizados;
- registrar a evolução do projeto ao longo da disciplina;
- facilitar a consulta e a revisão dos documentos;
- apoiar a organização dos materiais acadêmicos da equipe.

## Observações

- Os nomes dos arquivos e pastas devem permanecer claros e padronizados.
- Novos documentos devem ser adicionados preferencialmente na pasta `Documentos/`, salvo necessidade de uma organização diferente.
- O repositório pode ser expandido conforme a evolução da disciplina e dos entregáveis.
